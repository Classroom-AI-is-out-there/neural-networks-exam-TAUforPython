# Учебное задание 2.2: Обучение без учителя - кластеризация PPG-сегментов по типу физической активности

## Задача

> **У вас есть датасет** коротких окон wearable-сигналов, содержащих фотоплетизмограмму (`PPG`) (результат регистрации изменений, возникающих при наполнении мелких сосудов кровью в зависимости от фазы кардиоцикла). Для каждого окна известен тип физической активности, но **эти метки нельзя использовать при обучении** - это *задача обучения без учителя*.

> **Вы должны написать код для кластеризации сегментов PPG**, выполнив следующие шаги:
> 1. Загрузить подготовленный датасет `dataset/task2_2-ppg-activity-clustering.npz`
> 2. Получить векторные представления окон временных рядов без использования меток при обучении
> 3. Разбить окна на кластеры и проанализировать, насколько кластеры соответствуют разным типам активности
> 4. Оценить качество кластеризации и визуализировать полученные группы

**Ожидаемый результат**: решение, которое группирует окна wearable-сигналов в осмысленные кластеры без использования меток активности в процессе обучения и показывает, насколько хорошо эти кластеры совпадают с реальными типами движений.

---

## Описание датасета

Датасет подготовлен на основе `PhysioNet Pulse Transit Time PPG Dataset v1.1.0` и содержит компактный учебный subset по активностям `sit`, `walk`, `run`.

| Свойство | Значение |
|----------|-------|
| **Формат** | NumPy archive (`.npz`) |
| **Образцы** | 72 окна временных рядов (8 испытуемых × 3 активности × 3 окна) |
| **Длина окна** | 8 секунд (`4000` точек при `500 Hz`) |
| **Каналы** | `pleth_1` (PPG) |
| **Истинные метки** | 3 активности, только для оценки |
| **Файл** | `task2_2-ppg-activity-clustering.npz` |

Структура архива:
- `X` - массив формы `(72, 4000, 1)` — PPG-окна
- `y` - истинные метки классов, использовать только для оценки
- `label_names` - список названий активностей
- `subject_id` - идентификатор испытуемого
- `record_name` - имя исходной записи
- `meta_json` - метаданные датасета (частота дискретизации, каналы, и т.д.)

Активности:
- `sit`
- `walk`
- `run`

## Требования к коду

✅ **Код** (`.ipynb` или `.py`), содержащий:
- Загрузку и предобработку PPG-сигналов
- Получение эмбеддингов или признаков без использования меток
- Кластеризацию окон, например через `KMeans`, `AgglomerativeClustering`, `DBSCAN` или аналогичный подход
- Оценку качества кластеризации с помощью `ARI`, `NMI`, `silhouette score`

✅ **Краткий отчёт** (не более 2000 знаков):
- Как Вы получали представление PPG-сигнала перед кластеризацией?
- Почему Вы выбрали именно этот метод кластеризации?
- Какие активности разделились хорошо, а какие смешались?

✅ **Визуализация в коде**:
- 2D-проекция эмбеддингов или признаков (`PCA`, `UMAP`, `t-SNE`)
- Раскраска точек по найденным кластерам
- Отдельная визуализация с раскраской по истинным меткам для анализа качества
- Несколько типичных PPG-сегментов из разных кластеров

## Ограничения и рекомендации

- Нельзя использовать `y` при обучении модели или настройке кластеризатора
- Истинные метки разрешается использовать только после кластеризации для оценки качества
- Решение должно быть интерпретируемым: важно показать, как Вы получили представление сигнала и как сравнивали кластеры с реальными группами активностей
