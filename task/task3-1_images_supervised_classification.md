# Учебное задание 3.1: Обучение с учителем – классификация медицинских изображений (MedMNIST)

## Задача

> **У вас есть датасет** медицинских изображений из коллекции MedMNIST, предварительно обработанных до единого формата 28×28 пикселей. Набор данных содержит изображения гистопатологических срезов (PathMNIST), размеченные по 9 типам тканей. **Вы должны использовать метки классов при обучении** — это *задача многоклассовой классификации с учителем*.

> **Вы должны написать код, выполнив следующие шаги:
> 1. Загрузить и предобработать данные из библиотеки `medmnist` (использовать флаг `'pathmnist'`)
> 2. Реализовать свёрточную нейронную сеть (CNN) для классификации изображений
> 3. Обучить модель на тренировочной выборке с валидацией
> 4. Оценить качество модели на тестовой выборке с помощью метрик: Accuracy, F1-score (macro), матрица ошибок

**Ожидаемый результат**: рабочий код с комментариями, где реализована нейронная сеть, которая классифицирует медицинские изображения по типам тканей, используя только пиксельную информацию (без дополнительных признаков).

## Описание датасета (PathMNIST)

| Свойство | Значение |
|----------|----------|
| **Модальность** | Гистопатология (лимфатические узлы, рак молочной железы) |
| **Формат** | 2D изображения 28×28 пикселей, RGB |
| **Количество классов** | 9 |
| **Размер выборки** | ~100 000 (train: 89 996, val: 10 004, test: 7 180) |
| **Метки классов** | 0 – жировая ткань, 1 – фон, 2 – дебрис, 3 – лимфоциты, 4 – слизь, 5 – гладкая мускулатура, 6 – нормальная слизистая оболочка толстой кишки, 7 – строма, ассоциированная с раком, 8 – эпителий колоректальной аденокарциномы |
| **Загрузка** | `medmnist` (флаг `'pathmnist'`) |

Изображения уже приведены к единому размеру, нормализованы в диапазон [0,1]. Для улучшения сходимости рекомендуется дополнительная нормализация (среднее=0.5, стандартное отклонение=0.5).

## Требования к коду

✅ **Код** (`.ipynb` или `.py`), содержащий:
- Загрузку датасета MedMNIST с помощью `medmnist` (разделы train, val, test)
- Предобработку: преобразование в тензоры, нормализацию
- Определение архитектуры свёрточной нейронной сети (например: два свёрточных слоя с пулингом, затем полносвязные слои)
- Цикл обучения с логированием функции потерь и точности на валидации
- Раннюю остановку (early stopping) для предотвращения переобучения
- Оценку на тестовой выборке: Accuracy, macro F1-score, матрицу ошибок (confusion matrix)

✅ **Краткий отчёт** (не более 2000 знаков):
- Как Вы справлялись с дисбалансом классов (если он присутствует)?
- Какая архитектура CNN была выбрана и почему?
- Какие классы модель классифицирует хуже всего и какие могут быть причины?

✅ **Визуализация в коде**:
- Примеры изображений из обучающей выборки с подписями классов
- Графики изменения функции потерь и точности на train/val в процессе обучения
- Матрицу ошибок на тестовой выборке (нормированную или абсолютную)
- Несколько примеров неправильно классифицированных изображений

## Примечания

- Датасет загружается автоматически через `medmnist` при первом вызове (размер ~200 МБ).
- Для ускорения обучения можно использовать графический процессор (CUDA), если доступен.
- Базовую архитектуру можно взять из примеров для MNIST, адаптировав количество входных каналов (3 для RGB) и выходных классов (9).
- Рекомендуется использовать функции из `medmnist.Evaluator` для расчёта метрик, но можно реализовать и вручную.
